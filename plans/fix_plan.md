## 4. Тестирование и отладка круговой секвенции

**Проблема**: Логика `is_valid_run` сложна и может содержать ошибки для крайних случаев (например, последовательность «Д, К, Т, 2, 3»).

**Файлы**:
- `scripts/game_service.gd`
- (опционально) создать тестовый скрипт `test_validation.gd`

**Изменения**:

**4.1. Добавить юнит-тесты в отдельный файл** (рекомендуется для будущего поддержания). Пример структуры теста:
```gdscript
extends SceneTree

func _init():
    var gs = GameService.new()
    print("Testing circular runs...")
    test_run(gs, ["Д", "К", "Т", "2", "3"], "♥", true)
    test_run(gs, ["Т", "2", "3", "4", "5"], "♠", true)
    test_run(gs, ["К", "Т", "2"], "♦", true)
    test_run(gs, ["9", "10", "В", "Д"], "♣", false)  # не круговая, но валидная
    quit()

func test_run(gs: GameService, ranks: Array, suit: String, expected: bool):
    var cards = []
    for r in ranks:
        cards.append({"suit": suit, "rank": r, "isJoker": false})
    var valid = gs.is_valid_run(cards)
    assert(valid == expected)
```

**4.2. Проверить текущую реализацию `is_valid_run` вручную** и исправить, если обнаружены баги.

**Примечание**: Если тесты выявят ошибки, потребуется корректировка алгоритма (возможно, переписать логику с использованием модульной арифметики).

## 5. Уточнение порядка карт в пуле

**Проблема**: Необходимо убедиться, что `discard_pile[0]` является верхней картой.

**Файлы**:
- `scripts/game_service.gd`
- `scripts/main.gd` (где создаётся визуальное отображение пула)

**Изменения**:

**5.1. Добавить комментарий в `draw_from_deck` и `draw_from_discard`, описывающий структуру:**
```gdscript
# discard_pile – массив открытых карт в пуле.
# Индекс 0 – верхняя (последняя сброшенная) карта, индекс size()-1 – нижняя.
```

**5.2. Проверить, что UI отображает карты в правильном порядке** (верхняя карта должна быть видна полностью, нижние – частично). Если порядок обратный, потребуется инвертировать массив при отрисовке.

**5.3. В `draw_from_discard` используется `slice(0, index+1)`, что означает взятие карт от верха до выбранной. Это соответствует правилу «вместе со всеми картами, которые лежат поверх нее». Оставить без изменений.

**Примечание**: Если порядок окажется обратным, нужно будет изменить все операции с `discard_pile` (например, использовать `push_back` вместо `push_front`). По текущему коду (`discard_pile.push_front(discarded_card)`) новая сброшенная карта становится индексом 0, что правильно.

**Файлы**:

- Если игрок пытается сбросить карту без использования целевой, выводится ошибка.

## 10. Ничья при одной карте в пуле

**Проблема**: Реализация корректна, но при ничье раунд перезапускается через таймер, что может привести к визуальным артефактам (например, игроки видят сообщение о ничье, но игра продолжается до таймера).

**Файлы**:
- `scripts/game_service.gd`
- `scripts/main.gd` (визуальная обработка)

**Изменения**:
- Добавить явное отключение интерактивных элементов на время показа сообщения о ничье.
- Убедиться, что `start_round` вызывается только один раз.

**Примечание**: Не критично, но улучшает пользовательский опыт.

## 11. Сетевая синхронизация

**Проблема**: Возможна десинхронизация при разрыве соединения или неожиданных RPC‑ошибках.

**Файлы**:
- `scripts/network_manager.gd`
- `scripts/game_service.gd`

**Изменения**:
- Добавить механизм повторной синхронизации: если клиент обнаруживает, что его состояние расходится с состоянием хоста (например, по таймауту), запросить полное состояние через RPC.
- В `game_service.gd` добавить функцию `request_full_state()`, которую может вызвать клиент.

**Примечание**: Это улучшение выходит за рамки исправления правил, но повышает надёжность сетевой игры.
